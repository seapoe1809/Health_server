Connect other devices using QR Code

This guide allows you to connect other devices to your Darna Server. It does require you to have camera enabled to scan QR codes:

1. Open "camera app" on your iOS. When pointed to QR code at top of screen it should give you a link to click on. Click on link.

2. On the bottom ribbon of browser window on your iOS device, click on '⮹'. Then choose 'Add to home screen'.

3. This link will now allow you to launch the server directly from your home screen when connected to local WIFI where the server is enabled.

What are the tags in 'Upload' and what are the directories for?

Upload allows you to uload files from any other device or phone to upload files to the Darna.HI server. With drop down menu you could either tag it as a 'Health File' or a 'Regular File'.

1. Health File: It is any health document you would like to save for obvious reasons. Tagging it allows it to be catalogued to the right subdirectory. The Health subdirectories are 'summary' and 'ocr_files'.

2. Regular Files: Is any non health file that you would like to store on your health server. It gets stored in the main folder 'Health_files'. Please note however iOS health file, 'export.zip' is preferred to be stored via this tag as that allows Darna.HI to identify and run Grafana and ingester to prepare dashboards.

If 'Export.zip' file containing your iOS health kit is available in 'Upload' directory and tagged a 'Regular File', it unzips that file, initates Grafana and Influxdb, populates influxdb with your records.

3. The 'summary' folder is where you 'Health' tagged files are stored. When you want to make a secure zip file to share, all the files in 'summary' get zipped. The default password is '2023'. You could change it in the darna.py code if you wish.

4. The 'ocr_files' folder is where your 'Health' tagged files which are of pdf, jpg, png ext are stored. When Analyze-card is run it OCR's the files and stores it in a text dump in Darna_tesseract subdirectory.

5. The 'Darna_tesseract' folder is where your 'Analyze' card stores relevant files. Files such as deidentified records, Personalized Health intents are stored in this subdirectory.

What does 'INDEX IT' do?.

Im glad you asked. 'INDEX IT' is many functions mashed into one. Broadly it sends tagged files to the right subdirectory. It started as just 'rsync' function that would move files from 'Upload' folder to the main folder 'Health_files'. However it has morphed as the Darna project has evolved. It does the following:

1. Runs 'rsync' to move tagged files from 'Upload' folder to 'Health_files', 'summary' and 'ocr_files' folder. It additionally screens for health files with .PDF, .JPG, .PNG, .TXT files and keeps a copy in ocr_files. This allows Tesseract in 'Analyze-card' to read those documents.

2. Initiates 'Caffeine' to keep you computer awake so that your server doesnt sleep when you are away. You dont need to turn off lockscreen while caffeine is running.

3. If 'Export.zip' file containing your iOS health kit is available in 'Upload' directory as a tagged 'Regular File', it unzips that file, initates Grafana and Influxdb, populates influxdb with your records.

How to view your dashboards on 'View my Device Data' via Grafana.

This guide allows you to connect other devices to the Grafana Viewer via Darna Server. It does require you to upload your device data:

1. Open "Health app" on your iOS. Then download iOS health files: On Apple "Health app", click the profile icon to the top right, scroll all the way down, then choose "Export All Health Data", click "Export" and save the zip file to iOS Files folder.

2. On the Darna Server, choose 'Uploads' card, click 'choose file' on next page, navigate to iOS files folder and choose 'export.zip' and finally choose 'upload'.

3. Once the 'export.zip' is in the 'uploads' folder with 'Regular File' tag, click 'INDEX IT' card on the Darna Server. In the background, the Grafana and ingester will start. This could take time (around 5-10 min) as they could be upto 100,000 records to transfer.

4. Once the records are transferred, click on the 'View my Device Data'. This should open a new webbrowser.

5. Login using user: admin, password: health which are the default username and password

6. Once logged in click on '≡' located at top left. That is the 'Toggle Menu'.

7. Once on 'Toggle Menu', click 'Dashboards'. That will open all the active dashboards to choose from.

8. Click on any of them to view your data. If you dont see any data, you might have to start steps all over again.

What is Pie in Sky?

Pie in Sky is a optional card installer. It's a one click docker container installer that launches a static site to interact with the docker container.

1. Once set up, clicking in the card will launch the static site to interact with the docker container.

2. It does require docker to be installed on your computer.

3. Tailscale is highly recommended to be installed. It allows enterprise grade remote access to Darna Server. Once again we are not involved in development or maintenance of Tailscale. Tailscale additionally requires an iOS or Android app to be able to remote access your Darna.HI server.

4. We hope to add more cards in future to the Pie in Sky.

How to earn while you run your server? Why Monero?

In the realm of financial privacy, the use of cryptocurrencies like Monero can play a role in enhancing users' security. Monero is a privacy-focused cryptocurrency that provides anonymity and obfuscation of transaction details. This can be particularly relevant in healthcare contexts where financial transactions may be involved, such as payment for medical services or insurance claims.

Darna.HI believes that there natural synchronization between healthcare security under HIPAA and individuals' security underscores the importance of safeguarding personal health information. By adhering to HIPAA guidelines, healthcare organizations demonstrate their commitment to protecting patient data. Simultaneously, individuals must be proactive in safeguarding their own data, employing secure practices and recognizing their role in maintaining privacy. Additionally, leveraging privacy-focused cryptocurrencies like Monero can further enhance financial security within healthcare transactions.

2. XMR miner: Allows user to run XMRIG pool miner in a docker container. Please note it can slow your computer
